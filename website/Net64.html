Net64 2.0

Net64 is a tool created by Kaze Emanuar & Melonspeedruns. It makes Super Mario 64 possible to play online with other players, making for a fun multiplayer experience. Since then, the project turned into Net64+ which is a fork of the original Net64 developed by Tarnadas. Which offers many new improvements to the client such as a serverlist.

FAQ



Q: Why does my emulator only show a white screen?
A: Refer to this: https://goo.gl/ecxeKW


Q: Why is the server separated from the client?
A: Because a clear client/server separation is always better and adding hosting possibilities to the client feels redundant. There will most likely always be 24/7 servers, but if you still want to host your own server for your friends or a LAN network or if you also want to host a 24/7 server, you should read the hosting guide.

However, if you choose to use the client in #setup-and-help called x86-32bits then you can use the built in server hosting like the old 1.3.1 hotfix client.


Q: How can my friend connect to my server?
A: Your friend needs your public IP address and the port you are using for the server must be forwarded.

How to do port forwarding is out of the scope of this tutorial, but if you just want to check, if your server is working, start it and after starting you can use various websites to check your port forwarding like this: https://portchecker.co/check Your public IP address does not look like this: 127.x.x.x, 192.x.x.x


Q: LAN server?
A: Instead of your public IP address, you need your local network IP address.

• press Win and type "cmd" + Enter • run "ipconfig" (without quotations) • look for your ethernet adapter and its IPv4 address and let your friend use this one

Do not forget, that you can always connect on the same machine with the IP address 127.0.0.1


Q: Hamachi?
A: Since Hamachi is just simulating a LAN, you just have to connect to the same Hamachi network and use your Hamachi IP address. Remember to only use Hamachi with those you trust.


Q: I’m invisible?
A: This is a common issue, it can be a lot of different things but most of the time it is because the server you connected to did not properly port-forward or allowed the software through the firewall. Other times it could be that you did not connect properly.

If it is not a server issue, I would double-check with another server. I will list some of the common fixes you can try but they may/may not work:

• Change your emulators graphical plugin

• Project64 1.6 (sometimes 2.3 as well) requires you to press F5 then F7 while on the file select screen after you have connected with the online tool. This creates a savestate and reloads it.

• Make sure your anti-virus did not delete anything from the online tool software

• Make sure your ROM is clean, if you look for it on the internet you want one called “Super Mario 64 (U) [!]” in .z64 format (if it is in a .zip or .rar you’ll need to extract it). NOTE: Europe (E) and Japanese (J) ROMs do not work with the online tool.

• Verify where you downloaded the online tool, some people tend to download it from different websites which we suggest you don’t do. On the discord server always get it from the #setup-and-help channel.

Always make sure to restart the tool and the emulator when you apply changes.


Q: How do I use a ROM hack like Star Road?
A: It includes a readme file that covers just about everything, I also created a video if you would like to manually make a ROM hack not listed work but it will require some trial and error but the process is easy enough to understand.

https://www.youtube.com/watch?v=jVsZbDZuSig While the video says it is for the 1.3.1 hotfix client of the online tool, the same rules and principle apply to Net64+ and the 86x-32bits client resources folder. Remember, read the readme included. Full ROM Hack support is planned in 2.0


Q: What’s the difference of x64 and x86-32bits of the tool in #setup-and-help?
A: Without going too deep into it, the x64 client of Net64+ features a full GUI and a way to find public servers much easier it was developed by Tarnadas. However it only works with PC systems that are of 64bits, it will not work at all on 32bit systems.

The x86-32bits client is based off purely of the original client but it connects to Net64+ servers instead and will work on all x64 and x86 systems so if it didn’t work at all before then this one will. You may find this client simpler to use if you prefer the original 1.3.1 hotfix client.


Q: Why is Windows Features blank?
A: If you don’t see anything on Windows Features when you enable “.NETFramework” and you cannot download “.NETFramework” then you’ll need to open Services.msc by pressing Win+R to open the run menu and typing Services.msc

Press enter and scroll down until you find Windows Modules Installer.

Right click and click on properties, set startup type to “manual”. From there restart your PC and go back into Windows Feature to re-enable “.NetFramework”.

Note: Be sure you are using the latest version of .NETFramework; 4.7 and up


Q: Do I download the source code?
A: No, you need to download the 7z above the source code download. This is the case for both the client and precompiled server downloads.

A 7z file is similar to a zippered folder (zip file), but you might need to download a program in order to open one. Popular programs that can open a 7z file are 7Zip and WinRAR.


Q: Why can’t I see other players?
A: Players can be anywhere in the game. Chances are that you'll have to go look for them. You can try asking them to go to a specific location, like the bridge at the castle’s front.

If you're absolutely certain that you're not seeing other players, you might need to perform an extra step when connecting to the server. Restart your emulator and Changing Emulator (use Navigation in the top-left), then reconnect to the server. Before selecting a file, press F5, then press F7.

If you still can't see other players, the server might be malfunctioning. Try a different server. If another server works, let the host of the malfunctioning server know about your problem.

If no servers seem to work, your emulator or Net64+ might be malfunctioning. At this point, it is recommended to start from square one, reinstall Project64, and redownload Net64+.


Q: How do I port forward?
A: Every router is different, so there is no simple universal answer to this question. You’re better off searching how to port forward for your particular brand and model of router.

1. Find the brand and model of your router. Both of these usually can be found somewhere on your router.

2. Use a search engine (such as Google) to search how to access your router's settings and how to set up a port forwarding rule.

For most, accessing your router's settings is done by entering a local IP (usually 192.168.0.1) or accessing a special website.

Port forwarding settings are commonly located in the Advanced Settings or Firewall section.

1. For the port forwarding rule, enter 3678 as the port, and use the TCP protocol.

If you can only enter a range, put 3678 as both the start and end. If the numbers must be different, set the end as 3679. Make sure to enter both internal and external ports, set both to 3678.

1. For the local IP, open the command prompt from the Start menu (type cmd), then type ipconfig in the command prompt.

a. If using a wired connection, look for a section with Ethernet Adapter in it. b. If using a wireless connection, look for a section with Wireless Adapter in it. c. If you do not see a section which has these in its name, use your best judgement. Your computer won't explode if you get the local IP incorrect, so just keep trying until it works. In the appropriate section, look for a row labelled IPv4 address. This is the local IP you should enter for the port forwarding rule.

1. The name of the port forwarding rule can be anything, but you should name it something you can remember, such as Net64plus.

2. There might be other settings for your port forwarding rule that might or might not be relevant. If you are unsure, ask us in #support-all-issues on the Net64 discord. Screenshots will help.

3. After your port forwarding rule is created, DO NOT FORGET TO ENABLE THE RULE!!!

4. If your server still does not work, before asking us for help, triple-check your port forwarding rule. If it's your first time, chances are that you missed something. If it isn't, you might want to check if your local IP (the one from ipconfig) changed.

5. After knowing for certain that everything is correct, if the server still does not work, ask us for help in #support-all-issues on the Net64 discord. Screenshots will help.
